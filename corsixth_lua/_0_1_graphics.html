Graphics Class

Layer for loading (and subsequently caching) graphical resources.

Declared on: line 25 of Lua/graphics.lua.
Full description:
The Graphics class handles loading and caching of graphics resources. It can adapt as the API to C changes, and hide these changes from most of the other Lua code.

function Graphics:loadFont(sprite_table, x_sep, y_sep, ttf_col, force_bitmap)

Load a font for the given sprite table.

Parameters:

NameTypeDescription
sprite_table userdata The sprite table of the font this font is replacing. Used for default colour and metrics.
x_sep integer Controls the horizontal separation of the font for bitmap fonts.
y_sep integer Controls the vertical separation of the font for bitmap fonts.
ttf_col colour or in or form or red or green or and or blue The colour of the font if a true type font is loaded. If nil, the colour will be detected from the sprite table.
force_bitmap boolean Whether to force the font to be a bitmap font.

function Graphics:loadFontAndSpriteTable(dir, name, complex, palette, x_sep, y_sep, ttf_col)

This is the same as calling loadFont(Graphics:loadSpriteTable(dir, name, complex, palette), x_sep, y_sep, ttf_col)

except that force_bitmap is set to true automatically for some sprite tables.

Parameters:

NameTypeDescription
dir string The directory of the sprite table relative to the HOSPITAL directory.
name string The name of the sprite table file without the .dat extension.
complex boolean Whether the sprite table is complex or not.
palette string The name of the palette file. If nil, the default palette will be used.
x_sep integer Controls the horizontal separation of the font for bitmap fonts.
y_sep integer Controls the vertical separation of the font for bitmap fonts.
ttf_col colour or in or form or red or green or and or blue The colour of the font if a true type font is loaded. If nil, the colour will be detected from the sprite table.
